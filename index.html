What is TypeScript?

TypeScript is a typed superset of JavaScript that adds static types to improve code reliability and scalability. It compiles to plain JavaScript, running anywhere JavaScript runs.

  • Developed by: Microsoft
  • Version: 5.8 (stable), 5.9 (beta)
  • Benefits: Early error detection, enhanced tooling, scalability, gradual adoption.

Example: Early Error Detection

const user = { name: "Alice" };
console.log(user.age); // Error: Property 'age' does not exist

Explanation: TypeScript infers the user objectâ€™s shape as { name: string }. Accessing user.age triggers a compile-time error, preventing runtime issues.

Why Use TypeScript?

TypeScript enhances JavaScript development by providing compile-time type checking, which catches errors early, improves code quality, and enhances maintainability.

  • Error Detection: Catches issues before runtime.
  • Tooling: Autocompletion, refactoring, and better IDE support.
  • Scalability: Used by companies like Slack for large codebases.
  • Compatibility: Compiles to JavaScript, with types removed for seamless execution.
  • Gradual Adoption: Can be introduced incrementally in existing JavaScript projects.

Example: TypeScript to JavaScript

type Result = "pass" | "fail";
function verify(result: Result) {
  return result === "pass" ? "Passed" : "Failed";
}
// Compiles to:
function verify(result) {
  return result === "pass" ? "Passed" : "Failed";
}

Explanation: The Result type restricts result to "pass" or "fail". Types are removed during compilation, producing clean JavaScript.

How to Use TypeScript?

TypeScript can be set up and used with simple commands for installation, compilation, and configuration.

  • Install: npm install -g typescript
  • Check Version: tsc --version
  • Get Help: tsc --help
  • Compile Single File: tsc hello.ts
  • Watch Mode (Single File): tsc --watch hello.ts
  • Compile Multiple Files: tsc --watch hello.ts world.ts
  • Compile Directories: tsc --watch hello.ts world.ts src/**/*.ts
  • Initialize tsconfig.json: tsc --init
  • Compile with tsconfig.json: tsc --project tsconfig.json
  • Watch with tsconfig.json: tsc --watch --project tsconfig.json

Example: tsconfig.json

{
  "compilerOptions": {
    "target": "es6",
    "module": "commonjs",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src"
  }
}

Explanation: Configures the compiler to output ES6 JavaScript, use CommonJS modules, enable strict type-checking, and organize files in dist and src directories.

Static vs. Dynamic Type Checking

Static Type Checking: Performed at compile time, catching errors before execution. Improves code quality and maintainability.

Dynamic Type Checking: Performed at runtime, catching errors after code executes.

Example: Static Type Checking

let num: number = "hello"; // Error: Type 'string' is not assignable to type 'number'

Explanation: Assigning a string to a number-typed variable triggers a compile-time error, ensuring type safety before execution.

Type Annotations & any Type

Type annotations specify variable types. The any type allows any value but risks errors.

Example:

let x: number = 10;
let y: string = "Hello";
let z: boolean = true;
let a: any = "Hello"; // any data type

Explanation: x, y, and z are strictly typed, while a: any allows any value, bypassing type safety.

Arrays & Multidimensional Arrays

Type annotations specify array element types. Multidimensional arrays support mixed types via unions.

Example:

let arr: string[] = ["Ahmed", "Basant", "Ali", "Hoda"];
let arr3: (string | number | boolean)[] = ["Ahmed", "Mona", 23, 14, true, false];
let arr4: (string | number[] | boolean[])[] = ["Hazem", "Menna", [23, 132, 45, 54], [false, true]];

Explanation: arr is a string array, arr3 allows mixed types, and arr4 includes nested arrays.
