Functions

Functions support typed parameters and return types, with compiler options for strictness.

Example:

let showMsg = true;
function showDetails(name: string, age: number, salary: number): string {
    if (showMsg) {
        return `My Name is ${name}, My Age is ${age} and My Salary is ${salary}`;
    }
    return 'Thanks';
}
console.log(showDetails('Ahmed', 23, 1000));

Explanation: The function ensures type safety for parameters and return value. Compiler options like noImplicitAny enforce strict typing.

Default & Optional Parameters

Default parameters provide fallback values; optional parameters are optional.

Example:

function showDetails2(name: string, age: number = 20, salary?: number): string {
    return `My Name is ${name}, My Age is ${age} and My Salary is ${salary}`;
}
console.log(showDetails2("Basant"));

Explanation: age defaults to 20, and salary is optional, allowing flexible function calls.

Rest Parameters

Rest parameters collect arguments into an array using ....

Example:

function showDetails3(name: string, ...rest: number[]): string {
    return `My Name is ${name}, My Age is ${rest[0]} and My Salary is ${rest[1]}`;
}
console.log(showDetails3("Basant", 23, 1000));

Explanation: rest collects numbers into an array, ensuring type safety.

Anonymous & Arrow Functions

Anonymous functions use function expressions; arrow functions use => for concise syntax.

Example:

const showDetails4 = function(name: string) {
    return `My Name is ${name}`;
}
console.log(showDetails4("Basant"));

const showDetails5 = (name: string) => {
    return `My Name is ${name}`;
}
console.log(showDetails5("Basant"));

Explanation: Both functions return the same result, with arrow functions being more concise.
